El problema y la oportunidad
La adopción de IA en equipos de software en LATAM es superficial, desigual y sin estrategia. Las empresas que cambien eso primero ganan una ventaja que los demás no podrán alcanzar.
⚡ El problema actual
  • Uso disperso de IA — Copilot aquí, ChatGPT allá — sin estándar ni estrategia
  • Prompts inconsistentes que no se comparten ni evolucionan entre el equipo
  • Sin infraestructura: cada dev usa su cuenta personal, sin contexto del codebase
  • Resistencia cultural: devs seniors que sienten que la IA los reemplaza
  • Sin métricas: no hay forma de medir si la IA realmente está impactando
  • Equipos sobredimensionados haciendo tareas que la IA debería automatizar
🚀 La oportunidad
  • Reducir ciclo ticket-to-deploy en un 40–50% con agentes integrados al CI/CD
  • Eliminar el bottleneck de QA y documentación — 100% automatizados desde el día 1
  • Onboarding de nuevos devs en menos de 1 día — no semanas
  • El mismo equipo entrega 2–3× más features sin contratar más personas
  • Modelo de ROI medible y ejecutivo-ready desde el primer mes
  • La ventana de ventaja competitiva es ahora — 2025 es el año de separarse

🔧 Track Técnico — Infraestructura AI para tu equipo
Construimos y desplegamos la infraestructura técnica completa: desde el AI Dev Kit hasta el pipeline CI/CD con agentes. Impulsado por la plataforma Nébula de Andino Labs.

🎁 Entregable principal: AI Dev Kit

Un kit instalable en menos de 10 minutos en cualquier máquina del equipo. Conectado al codebase, integrado al flujo de git, y listo para impactar la productividad desde el día 1.

✓ CLI aix con comandos por rol integrados al workflow de git
✓ Prompt Library 30+ prompts curados por rol (Dev, QA, Tech Lead, PM)
✓ Context Engine — RAG conectado al codebase, docs y tickets del equipo
✓ Agentes de Code Review, Testing, Docs y Security en CI/CD
✓ LLM Gateway con rate limiting, cost tracking y model routing
✓ Dashboard de métricas: uso, aceptación de prompts, impacto en velocity
🏗 Semana 1–2 · Infraestructura base
  • Setup del monorepo ai-framework/: prompts/, agents/, context/, cli/, docs/
  • Selección y configuración del LLM Gateway (Azure OpenAI, Anthropic, o self-hosted)
  • Despliegue del vector DB: pgvector, Chroma o Pinecone según infraestructura
  • Pipeline de ingesta: codebase → chunking → embeddings → índice searchable
  • CI/CD secrets y variables de entorno configuradas
  • Definición del contrato de evaluación: ¿qué hace bueno a un prompt?
📝 Semana 2–3 · Prompt Library + CLI
  • Mob Elaboration: workshop de 2–3h donde AI propone descomposición de Intents, equipo valida
  • Construcción de 30+ prompts por rol con estructura: Role + Context + Instruction + Format + Examples
  • CLI aix: aix review, aix gen-test, aix explain, aix gen-feature, aix gen-docs
  • A/B evaluation de prompts: manual vs. AI-assisted (tiempo + calidad medidos)
  • Prompts publicados como artefactos YAML versionados en repositorio
🤖 Semana 3–4 · Agentes en CI/CD
  • Code Review Agent: lee el diff + estándares del equipo → comenta en el PR
  • Testing Agent: genera tests unitarios a partir del diff + contexto del módulo
  • Docs Agent: actualiza docs técnicos y changelog en cada merge a main
  • Security Agent: OWASP Top 10 scan en cada PR — bloquea findings críticos
  • Integración con GitHub Actions, GitLab CI o Azure DevOps
  • Métricas baseline capturadas: coverage delta, PR cycle time, review time
🚀 Mes 2+ · Agente de Scaffolding (Spec2Code)
  • Input: ticket + especificación → Output: archivos scaffold, tests, migraciones, API contracts
  • Integración con Jira o Linear: el agente lee el contexto del ticket automáticamente
  • Sigue los patrones del codebase existente — no genera código genérico
  • Framework v2: prompts afinados con 4–8 semanas de datos reales de aceptación
  • Nuevos devs productivos en menos de 1 día usando el framework

🧠 Track Cultura — El cambio de paradigma
Las herramientas sin el mindset correcto se abandonan en semanas. Este track convierte a cada persona del equipo en un orquestador de IA — con talleres, rituales y un programa de gestión del cambio diseñado por perfil.
El problema de fondo: Un dev senior con 10 años de experiencia siente que la IA lo hace menos valioso. Un QA cree que los agentes de testing lo van a reemplazar. Un CTO no sabe cómo medir el ROI de la IA. Sin abordar esto directamente, cualquier adopción técnica fracasa.
Programa por perfil
Cada rol tiene sus miedos, sus motivaciones y su relación con la IA. El track cultura diseña una experiencia específica para cada uno.
Desarrollador / Senior Dev
De "yo escribo código" a "yo diseño y orquesto IA"
  • Reencuadre: la IA no te reemplaza, multiplica tu impacto
  • Taller hands-on: tu primera Feature completa con AI Dev Kit en <2h
  • Práctica de Mob Construction: construir en equipo con IA como pair programmer
  • Cómo revisar código generado por IA con criterio de senior
  • Prompt engineering avanzado para tu stack específico
Tech Lead / Architect
Orquestar al equipo y a la IA en paralelo
  • Mob Elaboration facilitado por IA: cómo preparar y conducir la sesión
  • Diseño de Domain y Logical Design con IA — ADRs generados y validados
  • Cómo definir estándares de calidad para código generado por IA
  • Gestión de adopción en el equipo: champions, métricas, feedback loops
  • Governance: quién aprueba qué, qué gates son obligatorios
QA / Testing
El rol evoluciona — no desaparece
  • De escribir tests manuales a supervisar y mejorar el Testing Agent
  • Cómo definir criterios de aceptación que la IA pueda validar automáticamente
  • Evaluación de outputs del agente: qué validar, qué aceptar, qué rechazar
  • Diseño de estrategias de testing con AI para escenarios complejos
  • Métricas de calidad en el mundo AI-native: qué medir ahora
Product Owner / PM
Especificar en el lenguaje que la IA entiende
  • Cómo escribir Intents y User Stories que maximizan la calidad del output de la IA
  • Participación en Mob Elaboration: tu rol en la sesión de descomposición
  • Nuevos ciclos de entrega: Bolts en lugar de Sprints — cómo planificar
  • Métricas de delivery en equipos AI-native: velocity, throughput, calidad
  • Comunicación con stakeholders sobre la transformación
CTO / VP Engineering / Director de Tecnología
Construir la ventaja competitiva del equipo — y medirla
  • El modelo de organización AI-native: estructura, roles y accountability
  • Cómo presentar el ROI de la IA al board y a los clientes
  • Governance de IA: políticas de uso, seguridad, costos, compliance
  • Nuevas SLAs posibles con un equipo AI-native — propuesta de valor al cliente
  • Cómo liderar el cambio cultural sin perder a los mejores talentos
  • Roadmap de optimización de equipo a 6–12 meses con IA
Rituales AI-DLC
Los rituales son el corazón del cambio cultural. Reemplazan ceremonias lentas y desconectadas por sesiones intensas donde el equipo y la IA trabajan juntos.
🎯 Mob Elaboration
  • Duración: 2–3 horas (reemplaza semanas de planning)
  • Participantes: Product Owner, Tech Lead, Devs senior, QA
  • Dinámicas: IA propone descomposición de Intents → equipo valida y corrige
  • Output: User Stories, NFRs, Units y Bolts listos para construcción
  • El equipo y la IA quedan profundamente alineados antes de escribir una línea
  • Primera sesión facilitada por Andino Labs, luego el equipo la gestiona solo
⚡ Mob Construction
  • Duración: continua — reemplaza sprints de 2 semanas por Bolts de horas a días
  • Dinámica: dev establece sesión → IA guía Domain Design → Logical Design → código + tests
  • Revisión humana en cada paso — no es "IA hace todo", es "IA propone, humano aprueba"
  • Los Deployment Units salen testeados, documentados y seguros antes del deploy
  • Cada Bolt completado retroalimenta el Context Engine — mejora continua
📊 Retro AI-Driven
  • 30 minutos semanales con datos reales: tasas de aceptación de prompts, cycle time, coverage
  • IA analiza los datos y propone ajustes al Prompt Library y agentes
  • El equipo decide qué cambios implementar — datos guían, humanos deciden
  • Evolución continua: el framework mejora con cada iteración
🏆 Champions Program
  • 1 AI Champion por squad o área — referente interno para el resto del equipo
  • Capacitación avanzada: cómo mejorar prompts, evaluar agentes, entrenar al equipo
  • Canal de feedback directo con Andino Labs para escalar dudas y mejoras
  • Los champions construyen la capacidad interna para que la empresa no dependa externamente
📊 Track Consultoría — Optimización de equipo y procesos
Evaluamos el equipo actual con el lente de la IA: qué roles agregan valor diferencial, qué procesos se pueden automatizar, y cómo reestructurar para máximo performance y delivery.

🎁 Entregable: Plan de Optimización del Equipo

Un documento ejecutivo con el diagnóstico actual, el modelo de equipo optimizado para IA, el plan de transición y el modelo de ROI proyectado — listo para presentar al board.

✓ Current-state report: mapa de procesos, waste map, bottlenecks identificados
✓ Análisis de estructura: roles, seniority, distribución de trabajo por tipo
✓ Modelo de equipo optimizado: qué perfiles potenciar, cuáles redefinir
✓ Proyección de ROI: ahorro FTE equivalente o incremento de throughput
✓ Roadmap de transición: plan de 3–6 meses con hitos medibles
🔍 Fase 1 · Diagnóstico profundo
  • Auditoría de procesos: mapeo del ciclo completo ticket → PR → review → QA → deploy
  • Entrevistas estructuradas: 30 min por rol — Tech Lead, seniors, QA, PM
  • Análisis de métricas históricas: 90 días de cycle time, PR review time, defect rate, deployment frequency
  • Inventario de stack: lenguajes, frameworks, CI/CD, cloud, cobertura de tests
  • Análisis FTE: tiempo por rol — coding vs. review vs. docs vs. QA manual vs. reuniones
  • Output: Current-State Snapshot con waste map visual
📐 Fase 2 · Análisis de equipo
  • Evaluación de perfiles: qué hace cada rol y cuánto de eso puede hacer la IA
  • Identificar tareas automatizables: testing, code review, documentación, scaffolding — ordenadas por impacto
  • Análisis de seniority vs. valor: ¿el mix actual es el óptimo con IA?
  • Bottlenecks de delivery: dónde se acumula el trabajo, dónde está el mayor desperdicio
  • Output: Team Analysis Report con oportunidades priorizadas
💡 Fase 3 · Modelo optimizado
  • Estructura recomendada: equipo objetivo con roles redefinidos para el mundo AI-native
  • Nuevos perfiles emergentes: AI Champion, Prompt Engineer, AI QA Specialist
  • Redistribución de responsabilidades: qué hace el humano, qué hace la IA, quién revisa
  • Plan de upskilling: qué necesita aprender cada persona para el modelo futuro
  • Output: Target Operating Model para el equipo AI-native
📈 Fase 4 · Modelo de ROI
  • Cálculo de eficiencia: tiempo ahorrado por tarea × frecuencia × costo por hora
  • FTE equivalente: ¿cuántos FTEs libera la automatización para trabajo de mayor valor?
  • Throughput delta: ¿cuántas más features puede entregar el mismo equipo?
  • Costo vs. beneficio: inversión en el programa vs. valor generado en 6–12 meses
  • Output: ROI Model presentable a board y clientes

El proceso de diagnóstico
El diagnóstico evalúa las 9 dimensiones de AI Readiness en una sesión inicial de 90 minutos. Gratuito. El output es el AI Strategy Brief con el score por dimensión y el programa a medida.
📅 Sesión de diagnóstico (90 min)
  • Bloque 1 (30 min): Estrategia, governance y organización — entrevista con CTO/VP Eng sobre visión, riesgos y estructura del equipo (D1, D2, D5)
  • Bloque 2 (30 min): Tecnología, datos y delivery — revisión del stack, CI/CD, métricas y proceso actual (D3, D4, D6)
  • Bloque 3 (30 min): Seguridad, observabilidad y ecosistema — políticas, herramientas actuales, regulación relevante (D7, D8, D9)
  • Output inmediato: score por dimensión y recomendación de punto de entrada
📄 Output: AI Strategy Brief
  • Score de AI Readiness en las 9 dimensiones con visualización de gaps
  • Nivel de madurez actual (N0–N4) y brechas críticas identificadas
  • Punto de entrada recomendado y énfasis por track
  • Objetivos y métricas de éxito a 30/60/90 días
  • Estimación de ROI potencial para el equipo específico
  • Propuesta de programa personalizada lista para aprobar
